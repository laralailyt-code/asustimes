## 平台概述

**ASUSTIMES** 是一個為華碩及科技產業工作者設計的**即時情報監控平台**，提供：
- 🔴 **即時新聞聚合** — 來自 Bing News 和多個新聞源的科技產業新聞
- 📊 **原物料與匯率行情** — 實時追蹤關鍵材料和匯率走勢
- 🌐 **供應鏈風險評估** — 地緣政治、氣候、罷工等風險監控

**更新方式**：
- 新聞：自動每30分鐘刷新 | 手動刷新隨時可用
- 行情：每日 07:00/09:00/11:00/13:00/15:00/17:00 更新（台北時間）
- 風險：即時監控 | 基於過去 8 週（56 天）的新聞報導

## 🌐 供應鏈風險監控

### 監控事件類型

| 事件類型 | 監控週期 | 說明 |
|---------|---------|------|
| **颱風/洪水** | 近 3 天內 | 超過 3 天自動隱藏 |
| **地震** | 持續監控 | M≥5.0 顯示，但不計入風險評分 |
| **地緣政治衝突** | 過去 8 週（56 天） | 基於新聞發布日期判定 |
| **罷工** | 過去 8 週（56 天） | 存檔記錄；供應商頁面僅顯示 7 天內新聞 |
| **停電/限電** | 即時偵測 | 新聞關鍵字監控 |

### 事件嚴重程度
```
🔴 CRITICAL — 最嚴重（強颱、大地震、戰爭）
🟠 HIGH — 高風險（颱風級、M6+ 地震）
🟡 MED — 中等風險
🟢 LOW — 輕微風險
```

### 風險指數
```
🔴 ≥60 — 立即關注，風險高
🟡 ≥28 — 持續監控
🟢 <28 — 正常運營
```

### 時間篩選
```
近一週 / 近二週 / 近四週 / 近八週
→ 根據新聞發布日期篩選事件
```

> ⚠️ **重要**：
> - 颱風/洪水**僅顯示 3 天內**的事件
> - 罷工、地緣政治事件：基於過去 **8 週**新聞
> - 供應商頁面罷工新聞：僅顯示**7 天內**有新聞的事件

### 數據解讀技巧

**原物料價格走勢**：
- 紅色漲→綠色跌→紅色漲 = 波動性高
- 平穩曲線 = 價格穩定
- 急速上升 = 短期成本壓力

**風險指數閱讀**：
- 🔴 紅燈 = 考慮多元化供應商、庫存提升
- 🟡 黃燈 = 持續監控、建立預案
- 🟢 綠燈 = 正常運營

**新聞篩選**：
- AI 摘要優先級 > 時間
- 多個來源報導 = 事件重大

## 常見問題 (FAQ)

### Q: 為什麼 Digitimes 新聞來自 Bing News？
**A**: Render 伺服器的 IP 被 Google 封鎖（HTTP 503），現已改用 Bing News RSS 聚合 Digitimes 文章。無需登入，自動更新。

### Q: 為什麼某個原物料沒有更新？
**A**: 可能原因：
1. 來源網站維護
2. 數據延遲（LME 有時差）
3. 系統使用上次已知價格

### Q: 颱風為什麼消失了？
**A**: 颱風/洪水**僅顯示 3 天內的事件**。超過 3 天自動隱藏。

### Q: AI 摘要為什麼有時沒有？
**A**: 
1. 該分類新聞太少（少於 2 篇）
2. AI 服務暫時不可用
3. 分類首次載入（等待背景處理）

### Q: 為什麼罷工新聞在某些地方會消失？
**A**: 
- **事件存檔**：基於新聞發布日期，超過 56 天會隱藏
- **供應商頁面**：特別篩選，僅顯示過去 7 天內有新聞的罷工事件
- 這是為了避免顯示過期信息

### Q: 我可以自訂監控的商品嗎？
**A**: 目前不支持自訂。若需監控特定商品，請反饋至開發團隊。

### Q: 鎢粉價格加載很慢，怎麼辦？
**A**: 鎢粉使用 SMM 網站的 Playwright 爬蟲，首次加載較慢。系統會每日自動更新，持續優化中。
