### 🔄 Carry-forward 機制

CSV 寫入時自動補洞，避免顯示空白：

| 規則 | 說明 |
|------|------|
| **向前沿用 (carry-forward)** | 空白格用「最近一筆真實值」沿用，標 `*`（如 `27000*`）|
| **向後回填 (carry-back)** | 第一筆真實點之前的空白用該真實值反向填入，**只限近 30 天內** |
| **真實值寫入時** | 自動覆蓋 `*` 沿用值 |
| **超過 30 天的舊歷史** | 不做 carry-back，保持空白（避免污染歷史趨勢圖）|

| 類別 | 圖示 | 來源 | 顯示時間窗口 |
|------|------|------|-------------|

### ✊ 罷工監控目標公司（8 家）

三星電子、富士康、SK海力士、LG、比亞迪、台積電、聯發科、UPS

### 🌐 地緣政治監控地區

中東/紅海、東歐戰爭、東亞（台海、朝鮮）、南亞（印巴）、中非（剛果鈷產區）、紅海航運、伊朗局勢

### 🛡️ 主語驗證機制

避免「事件被錯誤歸屬給只是被順帶提及的公司」：
1. **公司主語驗證**：標題 + 句子級共現 + 30 字鄰近窗口
2. **嚴格罷工關鍵字過濾**：必須含「罷工 / strike / walkout」等動作詞
3. **排除非勞資爭議**：法律糾紛 / 罰款 / 專利等不算

### 📊 風險指數計算（cluster_risk）

> 詳見 `DEVELOPER_GUIDE.md` 的「風險指數計算公式」章節。

**基礎權重**：
- 🌪 災害 = 30
- ⚔️ 地緣政治 = 20
- ✊ 罷工 = 20
- ⚡ 操作異常 = 15
- 💸 財務 = 0（不計分）

**修正係數**：
- 7 天內 × 1.0；第 8 天起每天遞減 10%（最低 0.3）
- 超過 21 天不計分
- 罷工/地緣政治「未確認」× 0.6
- 罷工/操作異常「< 7 天短期」× 0.7

**警示閾值**：
- 🔴 ≥ 60 立即關注
- 🟡 ≥ 28 持續監控
- 🟢 < 28 正常

## 4. Telegram Bot 推播

### 🤖 Bot 資訊

| 項目 | 值 |
|------|---|
| Username | `@Lalalaralara_bot` |
| 連結 | `https://t.me/Lalalaralara_bot` |
| 啟動模式 | Polling（本地）/ Webhook（M4 部署到 Render 後）|

### 🔄 更新與推播頻率

| 機制 | 頻率 | 說明 |
|------|------|------|
| 訂閱命中掃描 | **每 60 秒** | dispatcher 掃 `risk_events.notified=false` |
| 事件落地 | _do_strike_scan / _do_geo_scan 跑完即時 | 寫進 Supabase `risk_events` 表 |
| 推播限速 | 25 訊息/秒 | Telegram 全域上限 30/s，留 buffer |
| 同事件對同人 | 永遠只推 1 次 | 用 `notification_log` UNIQUE 約束 |

### 🔔 訂閱類型（4 種）

| 類型 | 範例 | 命中邏輯 |
|------|------|---------|
| **🌍 地區** | 「日本」、「台灣/新竹」 | 子字串雙向比對：訂「台灣」吃到事件「台灣/新竹」，反之亦然 |
| **📦 料件類別** | BATTERY / IC / MEMORY 等 20 種 | 事件地區內存在 supplier 含此 part_category |
| **🏭 供應商分布** | 從 22 筆既有點選 | 事件 region 與 supplier.region 雙向比對 |
| **📍 半徑** | (lat, lng, km) | Haversine 距離 ≤ km |

### ⏰ 推播新鮮度過濾

| 事件類型 | 推播窗口 | 原因 |
|---------|---------|------|
| **地震** | 發生 30 分鐘內 | 過了沒實際意義 |
| **其他事件**（罷工、衝突、災害、操作異常）| 24 小時內 | 當天通知才有用 |
| **沒有時間資訊** | 不過濾（保守放行）| 避免漏推 |

### 🚦 嚴重等級門檻

每筆訂閱可設最低觸發等級：
- 🟡 **低**（全收）：低/中/高都推
- 🟠 **中**（含以上）：中或高才推
- 🔴 **高**（僅高風險）：只推高風險

### 📋 推播訊息格式

```
🔴 高風險  ·  ✊ 罷工
[事件標題]

📍 地區：韓國
🕐 時間：2026-04-29 10:30 UTC
📦 影響：[供應鏈影響說明]
🔗 來源：[新聞連結]

📨 命中規則：地區訂閱：韓國

[📰 詳情] [🌐 開啟風險地圖]
[✅ 已知悉] [🔕 靜音此規則 24h]
```

附 `sendLocation` 顯示事件座標。高風險響鈴；低風險 `disable_notification`。

### 🛡️ 錯誤處理

- **403 Forbidden**（使用者封鎖 bot）→ 自動停用該使用者所有訂閱
- **429 Too Many Requests** → 退避重試
- **其他錯誤** → 記錄到 `notification_log.error_message`，不阻塞主流程

`telegram_users` / `subscriptions` / `notification_log` / `risk_events` / `suppliers` / `parts` / `supplier_parts` / `wizard_state`
